Under the hood

Six things worth ordering twice.

Decorator-pattern customization

Milk, syrup, whip, drizzle — every add-on wraps the order in its own class, recalculating price and description on the fly.

MVVM, all the way down

A single BrewViewModel exposes StateFlow for cart, totals, favourites, and order history — every screen reacts, nothing duplicates.

Survives the shift change

Jetpack DataStore keeps the cart, favourites, and order log intact across restarts — close mid-order, pick up right where you left off.

Sign in like the till

Baristas sign in with a 5-digit ID — their username and password in one. A fixed admin code and a guest mode cover the kiosk cases.

A receipt for every order

Placing an order drops a printable-style ticket with the full decorator chain, itemized totals, and a mock "print" button.

Composed, not just coded

Material3, Navigation Compose transitions, animated chips and a bouncing cart badge — built to feel like a real product, not a wireframe.
